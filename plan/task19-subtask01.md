## 리포트 구조

### 1. 요약 (Executive Summary)

- 테스트 목적 및 범위
- 전략별 검증 결과 요약 (통과/실패/조건부 통과)
- 최종 권장 전략

### 2. 환경 정보

- Kubernetes 클러스터 사양 (노드 수, CPU/Memory)
- 사용한 컴포넌트 버전 (decisions.md 참조)
- 테스트 토픽 파티션 수, Consumer 레플리카 수, Producer TPS

### 3. 시나리오별 결과

각 시나리오(1~5)에 대해:
- 시나리오 목적 및 수행 조건
- 측정된 지표 (전환 시간, 롤백 시간, 유실/중복, Lag)
- Grafana 스크린샷 첨부
- 검증 기준 통과 여부 및 이유

### 4. 전략별 비교표

| 항목 | 전략 B | 전략 C | 전략 E |
|------|--------|--------|--------|
| 실측 전환 시간 | | | |
| 실측 롤백 시간 | | | |
| 메시지 유실 건수 | | | |
| 메시지 중복 건수 | | | |
| Rebalance 발생 횟수 | | | |
| 구현 복잡도 | | | |
| 운영 복잡도 | | | |

### 5. 발견된 문제 및 해결 방안

### 6. 결론 및 권장사항

- 설계 문서 예상 결과 vs 실제 결과 비교
- 각 전략의 적합한 도입 시나리오
- 프로덕션 도입 시 추가 고려사항
